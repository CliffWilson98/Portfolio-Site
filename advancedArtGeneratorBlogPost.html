JavaScript Procedural Art Web App

As part of my advanced web apps course we are required to build a full stack web app which includes backend database operations. I have decided to create a website in which users are able to create art on a JavaScript canvas and then share what they have created for other users to upvote if they like. If you are not interested in creating anything of your own, then you can still browse the website to view what others have created. It is important to note that the method used to generate the art in this project is completely different from the JavaScript Pattern Generator that is already hosted on my site. The JavaScript Pattern Generator creates images by simulating cubes that bounce off of the edge of the canvas and leave behind a trail. In this project users will have control over the images that are drawn to the canvas by using a set of pre defined modules which are combined to create the images. An example of the type of art that is possible with this approach is the image on this page. This image is created by first using a module that will generate a collection of random points. A second module can then be placed which takes the set of points as input, and draws circles around those points. . Users will be able to control the size and color gradient of the generated circles. The reason that some circles appear more faded than others is because there are two distinct images of circles which have been blended together. I will work to add more modules that can be combined to great effect, but even with a small amount there is a wide range of images that can be generated.

Although my plan is subject to change, my initial vision of the web app involves three pages. One page will be for users to browse any images that other users have created. Another page will be for users to view their profile and the art that they have created. The final, and perhaps most important, page will be the one in which images are actually created. It is difficult for me to accurately predict the database requirements before the construction of the core features of the app has begun, but I believe that my database will be relatively simple. I will need to keep track of each user's username and password, along with the images that they have uploaded. For each uploaded image I will need to keep track of the amount of likes that it has and which users have liked it.

This project is heavily inspired by existing procedural art creators such as SPARTAN, Tilemancer, and Material Maker. All of those projects are fascinating to me and I enjoy the way that they allow users to create fascinating imagery without requiring any artistic ability.
